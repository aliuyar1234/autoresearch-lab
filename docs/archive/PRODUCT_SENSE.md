# PRODUCT_SENSE.md

## Primary persona

A solo researcher with a strong NVIDIA workstation who wants real progress from one GPU.

They are willing to trade generality for:
- speed,
- control,
- hackability,
- and local autonomy.

## Their jobs to be done

1. Start a research night without babysitting.
2. Know what changed and why in the morning.
3. Re-run or confirm promising ideas.
4. Preserve good ideas and near misses.
5. Extend the search surface without collapsing the system.
6. Run different campaigns without manually rebuilding everything.

## What feels good to this user

- one obvious command to start
- one obvious report to read
- one obvious place where experiments live
- one obvious place where champions live
- one obvious place where to add a new campaign
- no mystery state hidden in tools or chat logs

## What annoys this user

- “agentic” theater with little actual capability
- manual cleanup after every crash
- unclear experiment provenance
- giant dependency trees
- hidden configuration sprawl
- systems that are impossible to patch locally

## Product promise

The lab should make the user feel like they own a compact personal research organization on one machine.

## UX priorities

1. morning report quality
2. reliable runs
3. understandable artifacts
4. easy replay / reproduce
5. easy campaign extension
6. easy structured search extension
7. code-level proposal export for big ideas
